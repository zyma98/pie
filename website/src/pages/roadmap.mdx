## In progress

- **Apple Metal backend** <span className="r-tag r-runtime">runtime</span>  
  Enable GPU execution on Apple silicon via Metal for laptop development and demos.

- **Native C++ inference backend** <span className="r-tag r-runtime">runtime</span>  
  Lean, low‑latency server core; smaller footprint vs. polyglot runtimes.

- **PEFT adapters API (e.g., LoRA)** <span className="r-tag r-api">api</span>  
  Hot‑swap adapters per request/session with safe isolation.

- **Multi‑GPU (tensor & pipeline parallelism)** <span className="r-tag r-perf">performance</span>  
  Scale model size and throughput across devices.

- **4‑/8‑bit quantization** <span className="r-tag r-perf">performance</span>  
  Reduce VRAM while preserving quality; per‑layer policies.

- **KV‑cache quantization** <span className="r-tag r-perf">performance</span>  
  Larger batches and longer contexts via cache compression.

- **WASI Preview 3 migration** <span className="r-tag r-runtime">runtime</span>  
  Better asynchronous I/O, networking support.

- **Preemptive inferlet scheduling** <span className="r-tag r-runtime">runtime</span>  
  Fairness and SLOs under load; pause/resume primitives.

- **Expanded model support** <span className="r-tag r-runtime">runtime</span>  
  Broader LLM coverage across families and sizes.

- **Inferlet standard libraries (Go / C++ / Python / JS)** <span className="r-tag r-tooling">tooling</span>  
  Ergonomic helpers for common patterns; batteries included.

We welcome proposals—open a GitHub issue with your use case or RFC.
